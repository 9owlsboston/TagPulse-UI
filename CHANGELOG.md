## Unreleased

### Fixed

- **Table page-size changer was stuck at 20 on Tag Reads + Alerts (bug).** Selecting **50** or **100 / page** did nothing ŌĆö it snapped straight back to 20. Root cause: the tables passed `pagination={{ pageSize: 20 }}`, and a literal `pageSize` is a **controlled** prop, so AntD reverted every size-changer selection on the next render. Switched both [Tag Reads](src/pages/telemetry/TagReads.tsx) and [Alert History](src/pages/rules/AlertHistory.tsx) to the uncontrolled `defaultPageSize: 20` (plus explicit `showSizeChanger: true` and `pageSizeOptions: [20, 50, 100]`), so each table owns its page size and the changer takes effect. Regression tests in [TagReads.test.tsx](src/pages/telemetry/TagReads.test.tsx) + [AlertHistory.test.tsx](src/pages/rules/AlertHistory.test.tsx) drive the changer (20 ŌåÆ 100) and assert the extra rows render. `npm run check` clean.

- **Tag Reads ŌĆö Antenna / Temperature / Humidity columns (Sprint 64).** The Tag Reads page rendered neither the **Antenna** (`reader_antenna`, already typed on `TagReadResponse`) nor the **Temperature / Humidity** values carried in the free-form `sensor_data` payload. Added three default-visible columns to [src/pages/telemetry/TagReads.tsx](src/pages/telemetry/TagReads.tsx): **Antenna** (keyed `reader_antenna`, next to Signal), **Temp (┬░C)** (`sensor_temperature`), and **Humidity (%)** (`sensor_humidity`). Temperature and humidity live under **two key conventions** ŌĆö real edge devices (wire-format v2) write `temperature_c` / `humidity_pct`, while simulators and demo seeds write `temperature` / `humidity` ŌĆö so a shared resolver reads a **fallback chain** (`temperature ?? temperature_c`) reused by the cell render, the `sorter`, and the CSV export; a numeric type-guard handles the `unknown`-typed dict values and renders `ŌĆö` when absent. All three are normal addressable columns (not `advanced`), so they flow through the ADR-032 `columns.tag_reads` config + the Sprint 63 column chooser and are included in the **Export rows CSV** (`reader_antenna`, `temperature_c`, `humidity_pct`). Tests in [TagReads.test.tsx](src/pages/telemetry/TagReads.test.tsx) cover the headers, both key conventions, and the `ŌĆö` fallback. No backend or contract change (UI-only). `npm run check` clean.

- **Tag Reads ŌĆö `EPC (hex)` column for hex-preferring tenants (chore).** The Tag Reads page rendered only the decoded **EPC URI** (`urn:epc:id:sgtin:ŌĆ”`) and `Tag ID`; the raw wire-format **`epc_hex`** (already in the `TagReadResponse` payload) had no column. RFID-only fleets (e.g. WM) that think in raw hex had to read the URI or rely on `Tag ID` coincidentally carrying the hex. Added an **`EPC (hex)`** column ([src/pages/telemetry/TagReads.tsx](src/pages/telemetry/TagReads.tsx)) keyed `epc_hex`, grouped next to `EPC`/`Scheme`. It's a normal addressable column (not `advanced`), so its visibility is **config-driven** via the ADR-032 `columns.tag_reads` leaf and the Sprint 63 column chooser: the backend system default hides it (so the readable URI stays the default ŌĆö no clutter), and a tenant/user that prefers hex reveals it (and hides the URI) via their `columns.tag_reads` preset or the "Columns" chooser. Test in [TagReads.test.tsx](src/pages/telemetry/TagReads.test.tsx) asserts the column renders. Pairs with the backend chore (system-default hide + WM tenant preset). `npm run check` clean.

- **Sprint 63 Tier 2 ŌĆö cross-device column visibility: server-backed "Columns" chooser + "Reset to team default".** A user's per-table column choices now follow them across devices and sessions instead of living in this browser's `localStorage`. The Sprint 62 device-local [`useLocalColumnVisibility`](src/lib/useLocalColumnVisibility.ts) hook is **retired** and replaced by [`useColumnVisibility`](src/lib/useColumnVisibility.ts), which reads the hidden set straight from the server-resolved `columns.<page>` leaf (`useColumnGroup`, fed by `GET /ui-config`'s SystemŌåÆTenantŌåÆRoleŌåÆUser merge) and writes through the new [ADR-032 v1.3](https://github.com/9owlsboston/TagPulse/blob/main/docs/adr/032-configurable-ui.md) endpoints: a merge-style **`PATCH /ui-config/me`** (so a column write composes with ŌĆö rather than clobbers ŌĆö the user's `cards`/`nav` prefs, even though there's no endpoint to read the user's own unmerged layer) plus a granular **`DELETE /ui-config/me/columns/{page}`**. The [`ColumnChooser`](src/components/ColumnChooser.tsx) gains a second, distinct reset: **"Show all"** (user override `hidden = []`, which *overrides* the team floor ŌĆö reveal every column) vs. **"Reset to team default"** (drop the user's `columns.<page>` leaf so the page re-inherits the tenant/role floor); a `busy` state disables the controls mid-write. The hot toggle path updates the resolved-config cache **optimistically** (a client mirror of the backend's list-replacing `deep_merge`), rolling back on error. New data hooks [`usePatchMyUiConfig`](src/hooks/useUiConfig.ts) + [`useResetMyColumns`](src/hooks/useUiConfig.ts); a new [`chooserCandidates`](src/lib/columnConfig.ts) helper lists the *full* addressable (non-advanced) set so a currently-hidden column still appears unchecked and can be re-shown. [Assets](src/pages/assets/AssetList.tsx) and [Tag Reads](src/pages/telemetry/TagReads.tsx) adopt the new hook; the [Preferences](src/pages/Preferences.tsx) **Save** switches from the wholesale `PUT` to the composing `PATCH` (so saving cards/nav no longer wipes the user's column overrides) while its **Reset** keeps `PUT {}` (the explicit clear-my-whole-layer verb). API types regenerated against the backend's updated `openapi.json` (PR [#107](https://github.com/9owlsboston/TagPulse/pull/107)). Tests: [useColumnVisibility.test.tsx](src/lib/useColumnVisibility.test.tsx) (toggle/show-all/reset payloads, no-op show-all, saving state), [useUiConfig.test.tsx](src/hooks/useUiConfig.test.tsx) (`deepMergeUiConfig` list-replace + optimistic PATCH cache update/rollback + reset invalidation), [ColumnChooser.test.tsx](src/components/ColumnChooser.test.tsx) (reset control + busy), [columnConfig.test.ts](src/lib/columnConfig.test.ts) (`chooserCandidates`), and [Preferences.test.tsx](src/pages/Preferences.test.tsx) (PATCH-save / PUT-reset split). `npm run check` clean (478 tests). Pairs with backend PR [#107](https://github.com/9owlsboston/TagPulse/pull/107); the regenerated `openapi.json` it consumes was built against that branch.
